Every day, IT departments are bogged down by a flood of repetitive support tickets like password resets and basic service status checks. I built the IT Support Triage Agent to automate these routine tasks safely, allowing IT professionals to focus on high-impact infrastructure challenges.

[0:20 — WHAT IT IS]
This agent belongs to the Agents for Business track. It is a secure, multi-agent workflow that classifies incoming IT requests, performs automated system diagnostics, resets user credentials under administrator oversight, and safely routes complex problems to human support tiers.

[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the workflow architecture. Every user request starts at the Security Checkpoint. If a security threat is detected, the workflow ends immediately to protect our backend systems. Otherwise, the clean query is routed to the Triage Orchestrator. The orchestrator determines whether the request requires diagnostics, an automated resolution, or human support. Diagnostics requests are routed to the Diagnostics Sub-agent, and resolution requests are routed to the Resolution Sub-agent. Both of these sub-agents execute tools through our custom MCP server. If a resolution requires a password reset, it triggers the HITL Admin Approval node, pausing execution until an administrator manually signs off. Finally, the results are formatted and presented back to the user.

[3:00 — SECURITY & MCP]
Our Security Checkpoint uses regex patterns to redact PII such as emails and IP addresses and scans for known injection keywords to write structured audit logs. The MCP Server runs locally via stdio, exposing check_system_status, reset_user_password, and restart_service tools to the sub-agents.
